This package contains classes used to provide a command language interface to the classes of the isb.chem package. For information on the isb.chem package, please refer to its javadoc pages. This command language permits defining a Model using a declarative command syntax that is very similar to a programming language: 

include "GlobalDefinitions.txt";
compartment "univ2": volume 10.0;
compartment "univ": volume 1.0;

species "s1": compartment "univ", speciesType boundary;        
species "s2": compartment "univ";
species "s3": compartment "univ";

parameter "k": value 1.0;                     

reaction "r1": reactants ("s1" "s2" "s2"), 
                         products ("s3"), rate "s1*s2*s2*k/2.0";   

speciesPopulations "sp": species ("s1" "s2" "s3"), 
                                   values (100 200 0);

model "myModel": reactions ("r1"), 
                           speciesMode molecules, parameters ("k"); 

printModel "myModel";
A full description of the command language can be found in the documentation for the CommandLanguageParser class.

This package can be compiled without installing the SBW system. However, in order to use the ScriptBuilderMarkupLanguage script builder class, you will need to have the SBW system installed on the build system and the target (runtime) system. In particular, this package makes use of the JAR files 

/usr/local/lib/SBWCore.jar
/usr/local/modules/SBWValidate.jar
whre "/usr/local" should be understood to denote the installation directory prefix where the SBW system was installed.
